Operator Classes¶

The classes are nested as follows:

nesting of the operator classes: solid lines means “has many”, dashed lines means “evolves into”¶

  • OperatorGrid

    • this is the master class which administrates all operator tasks

    • it is instantiated once for each run

    • it holds all necessary configurations

    • it holds all necessary instances of the Utility Classes

  • Operator

    • this represents a configuration for a fixed final scale \(Q_1^2\)

    • this performs the actual computation

    • this uses the 3-dimensional Operator Anomalous Dimension Basis

    • its OpMember are only valid in the current threshold area

  • PhysicalOperator

    • this is the connection of the Operator between the different flavor bases

    • it is initialized with the 3-dimensional Operator Anomalous Dimension Basis

    • it does recombine the operator in the Operator Evolution Basis (see Matching Conditions)

    • it exports the operators to Operator Flavor Basis in a ndarray

  • OpMember

    • this represents a single operator in Mellin space for a given element of the Operator Bases

    • inside Operator they are in Operator Anomalous Dimension Basis

    • inside PhysicalOperator they are in Operator Evolution Basis
