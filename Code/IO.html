Input & Output¶

For explaining the runcards we use the following convention:

  • key - type [default value, if available] : description

Input¶

The input run card may contain the following key-value pairs:

  • PTO - integer : order of perturbation theory: 0 = LO, …

  • alphas - float : reference value of the strong coupling \(\alpha_s(\mu_0^2)\) (Note that we have to use \(\alpha_s(\mu_0^2)\) here, instead of \(a_s(\mu_0^2)\) for legacy reasons)

  • xgrid - list(float) : the interpolation grid

  • xgrid_type - str : generating function for the interpolation grid

  • xgrid_polynom_rank - integer[4] : polynomial degree of the interpolating function

  • log_interpol - boolean [True] : use logarithmic interpolation?

  • q2_grid - list(float) : the target grid in \(Q^2\)

Output¶

The Output instance will contain the following key-value pairs:

  • xgrid - list(float) : the interpolation grid

  • polynomial_degree - integer : polynomial degree of the interpolating function

  • is_log_interpolation - boolean : use logarithmic interpolation?

  • q2_ref - float : starting scale

  • q2_grid - dict : all operators at the requested values of \(Q^2\) represented by the key

The grid elements contains two keys each

  • operators - dict : all evolution kernel operators where the key indicates which distribution is generated by which other one and the value represents the eko in matrix representation

  • operator_errors - dict : the integration errors associated to the respective operators following the same conventions as the operator dictionary

Example Output dump¶

is_log_interpolation: true
polynomial_degree: 4
xgrid: [0.001 0.01 0.1 1.0]
q2_ref: 1.0
q2_grid:
  1000.0:
    operator_errors:
      S.S:
        - [1 2 3]
        - [4 5 6]
        - [7 8 9]
      S.g:
        - [1 2 3]
        - [4 5 6]
        - [7 8 9]
      # more operators ...
  10000.0:
    operator_errors:
      S.S:
        - [1 2 3]
        - [4 5 6]
        - [7 8 9]
      S.g:
        - [1 2 3]
        - [4 5 6]
        - [7 8 9]
      # more operators ...
  # more q2 values ...

Note that by default PyYAML will put each element of the matrices on a new line - instead here we have chosen the more compact inline list representation for a better overview.
